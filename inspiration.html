FR-3 · Day Plan

Allow custom (non-task) items in the day plan + via CLI

"It also allows me to put custom tasks or text there. It doesn't necessarily have to be a task that I drag in there. I can also allow the user to click on the day plan and add something custom. It doesn't have to be just tasks."

The day plan schema (schedule[]) already supports block types beyond task — event, break, lunch, wake, other. But there's no UI to add these types — currently you can only drag tasks from the sidebar. Two changes needed: (1) click an empty time slot in the day plan to add a custom block (just text + duration, no task linkage); (2) expose this via the HTTP API so the CLI / Telegram bot can post a structured day plan with mixed task and non-task blocks.

Why this came up: the Telegram bot tried to send a day plan with a custom schedule and got back: "The /day tab doesn't support a custom schedule format via CLI — it shows MIT/P1/P2 with the task blocks pulled from notes. That's the limit of what the CLI exposes right now." The CLI / API needs POST /api/v1/day-plans/:date to accept a schedule array of mixed-type blocks, and the day-timeline UI needs a "click empty slot to add custom block" affordance.
FR-4 · Today Page

Redesign the MIT/P1/P2 task picker dropdown

"With 4 I meant that when you click the mit, p1, p2 empty row, it gives a dropdown of tasks. That currently does not look clean and it is overwhelming. The grouping or formatting of that needs to be better."

When you click an empty MIT/P1/P2 slot, a TaskPickerDropdown opens (currently rendered via portal in priorities-checklist.tsx).
