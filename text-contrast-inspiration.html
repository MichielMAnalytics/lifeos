Text Contrast — Problem & Fix

The dashboard text is hard to read due to overly dark muted colors and excessive opacity reductions stacked on top. Here are the options to fix it.

1. The Problem

Double opacity hit

The CSS variable --color-text-muted is already set to a dark color (e.g. #666666 on Midnight theme).

Then Tailwind classes like text-text-muted/40 apply 40% opacity on top — making the text nearly invisible.

Result: effective color on Midnight = #666666 at 40% opacity on #000000 bg = #292929 — a 1.4:1 contrast ratio (WCAG requires 4.5:1).

4. Fix Strategy — Two Options

Option A — Sync Colors + Cap Opacity

Recommended

1. Fix globals.css — sync --color-text-muted values to match themes.ts (brighter values)

2. Bulk replace low opacity — find-replace all /30 → /60, /40 → /70, /50 → /80

3. New minimum rule — no text below /60 opacity on text-muted

Pro: Quick, systematic fix. Touches many files but each change is mechanical.

Option B — Three-Tier Text System

Alternative

1. Define 3 text tiers in CSS

• text-primary — main content (current text)

• text-secondary — labels, metadata, hints

• text-tertiary — placeholders, disabled, decorative

2. Replace all text-muted/XX with the appropriate tier class

Pro: Cleaner architecture. No opacity hacks. Easier to maintain.
Con: Larger refactor — 351 instances to classify manually.
